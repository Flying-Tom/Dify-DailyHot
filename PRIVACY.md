# Privacy Policy

**Last Updated:** December 2025

## Overview

This Dify plugin ("Daily Hot") is designed to fetch trending data from publicly available sources via [DailyHotApi](https://github.com/imsyy/DailyHotApi). We respect your privacy and are committed to transparency about how this plugin operates.

## Data Collection & Usage

### What We Collect

- **API Requests**: Your plugin usage patterns (which sources you query, frequency)
- **Configuration Settings**: Plugin parameters you configure in Dify

### What We Don't Collect

- Personal information (name, email, location, etc.)
- User credentials or authentication tokens
- Content from your Dify workflows
- Any data beyond what's necessary to serve requests

## Third-Party Services

This plugin integrates with:

- **DailyHotApi** (<https://github.com/imsyy/DailyHotApi>): Public trending data aggregator
  - Please review [DailyHotApi's privacy practices](https://github.com/imsyy/DailyHotApi)
  - All data fetched is from public sources

## Data Storage

- **No data persistence**: This plugin does not store user data
- **Caching only**: Temporary caching of API responses for performance (configurable)
- **Your control**: Use `cache=false` to bypass caching

## Data Security

- All API communications use HTTPS
- No sensitive data is transmitted
- Plugin runs within your Dify environment

## Your Rights

You have the right to:

- Disable caching at any time
- Stop using the plugin
- Request information about plugin operations

## Changes to This Policy

We may update this policy occasionally. Changes will be reflected in this file with an updated "Last Updated" date.

## Contact

For privacy concerns or questions, please open an issue on the [GitHub repository](https://github.com/Flying-Tom/Dify-DailyHot).

**Note**: This privacy policy is for the Daily Hot plugin. Your Dify instance may have additional privacy policies that apply to your overall usage.
